HOW THE FEATURE WORKS (60 SECONDS):

1. User clicks ▶ on a tool → Form sheet opens
2. User fills parameters and clicks Execute
3. Sheet closes immediately (~100ms) — execution continues in background
4. User can start another tool while first one runs
5. Bottom panel shows all active/completed executions
6. User can click completed execution to see request/response details
7. User can retry failed executions

Result: Non-blocking, concurrent tool execution with visible queue.


KEY DESIGN DECISION:

✅ Bottom panel (persistent, resizable)
✅ Sheet closes immediately (non-blocking)
✅ Multiple tools run concurrently
✅ Detailed inspection (request/response in detail pane)
✅ Reuses existing Shipyard patterns (@Observable, @MainActor, sidebar+detail)
