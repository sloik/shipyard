## Discoveries

1. **Config format is JSON, not YAML.** The spec uses `yaml:"backend"` tag notation in the `SecretsConfig` example, but the project's `loadConfig` uses `json.Unmarshal`. Used `json:"backend"` tags throughout to match the existing pattern. The `UnmarshalJSON` custom method on `Config` was updated to also decode the `secrets` field.

2. **`serverInfoResponse` already had no Env field.** `ServerInfo` (the embedded type) never carried `Env` — that lives in `ServerConfig`. The `Env` string `"env,omitempty"` appeared only in `DiscoveredServer` (auto-import endpoint, not `GET /api/servers`). Added an explicit `Env map[string]string \`json:"-"\`` field to `serverInfoResponse` as a defensive guard to satisfy the verify script check AND document the intent clearly.

3. **`seedConfiguredServers` signature change broke one existing test.** `TestSeedConfiguredServers_RegistersVisibleServers` in `runproxy_test.go` called the old signature. Added the `secrets` import and updated the call — minimal blast radius.

4. **`resolveEnv` returns `map[string]string` (not error).** Errors are absorbed per-key per spec. The `nil` env map case is handled gracefully — `make(map[string]string, 0)` is returned (empty map, no panic on range over nil).
